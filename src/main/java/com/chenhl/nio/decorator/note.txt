1、定义

1.1、装饰器模式：又称包装模式。以对客户端透明的方式来扩展对象的功能（动态的给对象附加更多的责任），
换言之，客户端并不知道在装饰前与装饰后有什么不同。是继承关系的一种替代方案。
装饰模式可以在不创建更多子类的情况下，将对象的功能加以扩展。

1.2、装饰模式把客户端的调用委派给被装饰类。装饰模式的关键在于这种扩展是完全透明的。

1.3、装饰模式是在不必改变原类文件和使用继承的情况下，动态的扩展一个对象的功能。
它是通过创建一个包装对象，也就是装饰对象来包装真实的对象。



2、角色组成

-抽象构件角色（Component）: 给出一个抽象接口，以规范准备接受附件责任的对象。
-具体构件角色（Concrete Component）：定义一个将要接受附加责任的类
-装饰角色（Decorator）：持有一个构件（Component）对象的引用，并定义一个与抽象构件角色一致的接口
-具体装饰角色（Concrete Decorator）：负责给构件对象“贴上”附加的责任
